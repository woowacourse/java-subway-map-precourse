# 미션 - 지하철 노선도

> 우아한테크코스 : 프리코스 3주차 미션 '지하철 노선도'를 구현한 프로젝트입니다.
* 지하철 역과 노선을 관리하는 콘솔 프로그램입니다.
* 지하철의 역, 노선, 구간 등의 데이터에 대해 등록과 삭제 및 조회 기능을 제공합니다.
* 세부 프로그래밍 요구사항은 [java-subway-map-precourse](https://github.com/woowacourse/java-subway-map-precourse) 저장소의 README 파일을 준수했습니다.

<br>

## 구현 기능 목록

* [x] 메인 화면 UI를 출력하고, 사용자로부터 원하는 기능을 입력받는다.
    * 입력값이 Q인 경우, 프로그램을 종료한다.
    * 입력값이 1인 경우, 역 관리 기능으로 이동한다.
    * 입력값이 2인 경우, 노선 관리 기능으로 이동한다.
    * 입력값이 3인 경우, 구간 관리 기능으로 이동한다.
    * 입력값이 4인 경우, 지하철 노선도를 출력한다.
    * 그 외의 입력값은 에러 메시지를 출력한다.
* [x] 역 관리 기능 UI를 출력하고, 사용자로부터 원하는 기능을 입력받는다.
    * 입력값이 1인 경우, 등록할 역 이름을 입력받는다.
        * 다음 예외인 경우, 에러 메시지를 출력한다.
            * 지하철 역 이름은 공백이거나,  2글자 미만인 경우.
            * 이미 등록된 지하철 역들의 이름과 중복되는 경우.
        * 정상인 경우, 지하철 역을 등록하고 성공 메시지를 출력한다.
    * 입력값이 2인 경우, 삭제할 역 이름을 입력받는다.
        * 다음 예외인 경우, 에러 메시지를 출력한다.
            * 등록되지 않은 지하철 역 이름인 경우.
            * 이미 노선에 등록된 지하철 역 이름인 경우.
        * 정상인 경우, 지하철 역을 삭제하고 성공 메시지를 출력한다.
    * 입력값이 3인 경우, 역 목록을 출력한다.
    * 입력값이 B인 경우, 메인 화면 UI로 돌아간다.
    * 그 외의 입력값은 에러 메시지를 출력한다.
* [x] 노선 관리 기능 UI를 출력하고, 사용자로부터 원하는 기능을 입력받는다.
    * 입력값이 1인 경우, 등록할 노선 이름을 입력받는다.
        * 다음 예외인 경우, 에러 메시지를 출력한다.
            * 노선 이름이 공백이거나 2글자 미만인 경우.
            * 이미 등록된 노선들과 이름이 중복되는 경우.
        * 노선의 상행 종점역과 하행 종점역의 이름을 입력받는다.
            * 상행 및 하행 종점역의 이름에 대해서도 유효성(2글자, 중복 제외) 검사를 진행한다.
        * 정상인 경우, 지하철 노선을 등록하고 성공 메시지를 출력한다.
    * 입력값이 2인 경우, 삭제할 노선의 이름을 입력받는다.
        * 다음 예외인 경우 에러 메시지를 출력한다.
            * 등록되지 않은 노선 이름인 경우.
        * 정상인 경우, 노선을 삭제한다.
    * 입력값이 3인 경우, 노선 목록을 출력한다.
    * 입력값이 B인 경우, 메인 화면 UI로 돌아간다.
    * 그 외의 입력값은 에러 메시지를 출력한다.    
* [x] 구간 관리 기능 UI를 출력하고, 사용자로부터 원하는 기능을 입력받는다.
    * 입력값이 1인 경우, 구간 등록을 위해 노선 및 지하철 역 이름을 입력받는다.
        * 존재하지 않는 노선인 경우 예외 메시지를 출력한다.
        * 지하철 역이 해당 노선에 등록되지 않은 경우 예외 메시지를 출력한다.
        * 순서를 입력받는다.
            * 다음 예외인 경우 에러 메시지를 출력한다.
                * 1 이상의 정수가 아닌 경우.
                * 상행 종점 지하철 역이 아닌데 순서가 1인 경우.
                * 하행 종점 지하철 역이 아닌데 순서가 노선의 마지막 위치인 경우.
                * 노선에 등록된 지하철 역 개수보다 더 큰 숫자인 경우.
        * 정상인 경우, 구간을 등록하고 성공 메시지를 출력한다.
    * 입력값이 2인 경우, 구간 삭제를 위해 노선 및 지하철 역 이름을 입력받는다.
        * 다음의 예외인 경우 에러 메시지를 출력한다.
            * 존재하지 않는 노선인 경우.
            * 지하철 역이 해당 노선에 등록되지 않은 경우.
            * 노선에 등록된 역이 두 개 이하인 경우.
        * 정상인 경우, 구간을 삭제하고 성공 메시지를 출력한다.
    * 입력값이 B인 경우, 메인 화면 UI로 돌아간다.
    * 그 외의 입력값은 에러 메시지를 출력한다.

**모든 관리 기능들의 공통 구현 사항**

1. 에러 발생시, 작업 중이던 관리 화면의 UI를 다시 출력하고 재입력을 받는다.
    * 예를 들어, 노선 등록에 실패한 경우 메인으로 되돌아 가는 것이 아니다.
        * 노선 관리 화면을 다시 출력하고 재입력을 받아야 한다.

2. 원하는 기능의 작업이 성공하면, 메인 화면으로 되돌아간다.
    * 조회(출력), 등록, 삭제 등.
