# 미션 - 지하철 노선도

> 우아한테크코스 : 프리코스 3주차 미션 '지하철 노선도'를 구현한 프로젝트입니다.
* 지하철 역과 노선을 관리하는 콘솔 프로그램입니다.
* 지하철의 역, 노선, 구간 등의 데이터에 대해 등록과 삭제 및 조회 기능을 제공합니다.
* 세부 프로그래밍 요구사항은 [java-subway-map-precourse](https://github.com/woowacourse/java-subway-map-precourse) 저장소의 README 파일을 준수했습니다.

<br>

## 구현 기능 목록

### 초기 설정
역, 노선 등을 다음과 같이 초기 설정을 해둔다.

>
```
 1. 지하철역으로 교대역, 강남역, 역삼역, 남부터미널역, 양재역, 양재시민의숲역, 매봉역이 등록되어 있다.
 2. 지하철 노선으로 2호선, 3호선, 신분당선이 등록되어 있다.
 3. 노선에 역이 아래와 같이 등록되어 있다.(왼쪽 끝이 상행 종점)
   - 2호선: 교대역 - 강남역 - 역삼역
   - 3호선: 교대역 - 남부터미널역 - 양재역 - 매봉역
   - 신분당선: 강남역 - 양재역 - 양재시민의숲역
 ```

### 지하철 역 관리
* [x] 지하철 역을 이름으로 등록한다.
    * 지하철 역 이름은 null, 공백이 아닌 2글자 이상의 문자이다.
    * 이미 등록된 지하철 역들의 이름과 중복될 수 없다.
* [x] 등록된 지하철 역을 이름으로 삭제한다.
    * 등록되지 않은 지하철 역 이름은 삭제할 수 없다.
    * 이미 노선에 등록된 지하철 역은 삭제할 수 없다.
* [x] 지하철 역 명단을 출력한다.

### 지하철 노선 관리
* [x] 지하철 노선을 등록한다.
    * 지하철 노선의 이름은 null, 공백이 아닌 2글자 이상의 문자이다.
    * 이미 등록된 지하철 노선들의 이름과 중복될 수 없다.
    * 등록할 때, 해당 노선의 상행 종점 및 하행 종점역의 이름 또한 함께 입력한다.
        * 두 종점 또한 등록된 지하철 역 이름이어야 한다.
        * 두 종점은 중복될 수 없다.
* [x] 등록된 지하철 노선을 이름으로 삭제한다.
    * 등록되지 않은 지하철 노선은 삭제할 수 없다.
* [x] 지하철 노선의 명단을 출력한다.

### 지하철 구간 관리
* [x] 노선의 구간을 등록한다.
    * 구간을 추가할 노선의 이름, 구간으로 등록할 역 이름, 해당 구간의 순서를 입력받는다.
    * 존재하지 않는 노선에 구간을 등록할 수 없다.
    * 존재하지 않는 지하철 역을 노선의 구간으로 등록할 수 없다.
    * 순서는 1이상 (구간으로 등록된 지하철 역 개수 + 1) 이하의 범위만 허용한다.
    * 구간으로 등록할 지하철 역이, 이미 해당 노선의 구간으로 등록되어 있는 경우 등록할 수 없다.
* [x] 노선의 구간을 삭제한다.
    * 구간을 삭제할 노선의 이름, 삭제할 구간(역) 이름을 입력받는다.
    * 존재하지 않는 노선에 대해 구간 삭제를 수행할 수 없다.
    * 삭제할 구간(역)이 해당 노선의 구간으로 등록된 상태여야 한다.
    * 노선에 등록된 구간(역)이 2개 이하인 경우, 구간 삭제를 진행할 수 없다.

### 지하철 노선도 출력
* [x] 지하철 노선도를 출력한다.
    * 노선의 상행 종점부터 하행 종점까지 연결된 순서대로 역 목록을 조회할 수 있다.

### 지하철 노선 관리 프로그램의 UI 입출력
* [x] 메인 화면 UI를 출력하고, 사용자로부터 원하는 기능을 입력받는다.
    * 입력값이 Q인 경우, 프로그램을 종료한다.
    * 입력값이 1인 경우, 역 관리 기능으로 이동한다.
    * 입력값이 2인 경우, 노선 관리 기능으로 이동한다.
    * 입력값이 3인 경우, 구간 관리 기능으로 이동한다.
    * 입력값이 4인 경우, 지하철 노선도를 출력한다.
* [x] 역 관리 기능 UI를 출력하고, 사용자로부터 원하는 기능을 입력받는다.
    * 입력값이 1인 경우, 등록할 역 이름을 입력받는다.
    * 입력값이 2인 경우, 삭제할 역 이름을 입력받는다.
    * 입력값이 3인 경우, 역 목록을 출력한다.
    * 입력값이 B인 경우, 메인 화면 UI로 돌아간다.
* [x] 노선 관리 기능 UI를 출력하고, 사용자로부터 원하는 기능을 입력받는다.
    * 입력값이 1인 경우, 등록할 노선 이름을 입력받는다.
        * 노선의 상행 종점역과 하행 종점역의 이름을 입력받는다.
    * 입력값이 2인 경우, 삭제할 노선의 이름을 입력받는다.
    * 입력값이 3인 경우, 노선 목록을 출력한다.
    * 입력값이 B인 경우, 메인 화면 UI로 돌아간다.
* [x] 구간 관리 기능 UI를 출력하고, 사용자로부터 원하는 기능을 입력받는다.
    * 입력값이 1인 경우, 구간 등록을 위해 노선 및 지하철 역 이름을 입력받는다.
        * 순서를 입력받는다.
    * 입력값이 2인 경우, 구간 삭제를 위해 노선 및 지하철 역 이름을 입력받는다.
    * 입력값이 B인 경우, 메인 화면 UI로 돌아간다.
* [x] 지하철 노선도를 출력한다.
* [x] 역, 노선, 노선도 출력시 데이터가 없으면 [INFO] 안내를 표시한다.

**모든 관리 기능(VIEW)들의 공통 구현 사항**

1. 관리 화면 UI 이동시 잘못된 입력값을 입력받으면, 에러 메시지를 출력하고 재입력을 받는다.
    * 역 관리 기능에서 4를 입력한 경우, 다시 입력받는다.

2. 1번 항목을 제외한 에러 발생시, 작업 중이던 관리 화면의 UI를 다시 출력한다.
    * 예를 들어, 노선 등록에 실패한 경우 메인 UI로 되돌아 가는 것이 아니다.
        * 노선 관리 UI를 다시 출력한다.

3. 원하는 기능의 작업이 성공하면, 메인 화면으로 되돌아간다.
    * 조회(출력), 등록, 삭제 등.

4. 원하는 기능의 작업이 성공하면, [INFO] $메시지 형식의 알림을 출력한다.

5. 원하는 기능의 작업이 실패하면, [ERROR] $메시지 형식의 알림을 출력한다.
