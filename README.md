# 지하철 노선도 미션
- 지하철 역과 노선을 관리하는 지하철 노선도 기능을 구현한다.

<br>

## 💻 프로그램 패키지 및 클래스 구성

* Application - 전체 프로그램 실행 클래스
* SubwayApplication - 지하철 노선도 프로그램 실행 클래스

#### 패키지 구성

1. **userinterface 패키지** - 사용자 입출력 및 기능 선택시 Service를 수행하도록 하는 사용자 인터페이스 패키지

   a. mainmenu - 메인 메뉴 화면 클래스로 구성

   b. stationmenu  - 역 관리 메뉴 화면 클래스로 구성

   c. linemenu - 노선 관리 메뉴 화면 클래스로 구성

   d. intervalmenu - 구간 관리 메뉴 화면 클래스로 구성

   e. Input/Output controller 추상 클래스 - 사용자 입출력 및 정보 화면 출력 담당 controller

   f. Menu/MenuView 인터페이스 - 메뉴 관리 화면과 각 메뉴 클래스가 상속하는 interface

   g. ReturnController 클래스 - '돌아가기' 기능 클래스

2. **domain 패키지** - 프로그램 도메인 모델 및 repository 클래스로 구성 (Line, Station, LineRepository, StationRepository)

3. **service 패키지** - 각 메뉴들의 실제적인 기능을 수행하는 service 패키지 

   a. MainService - 메인 화면 기능 수행 클래스 (메뉴 선택 기능) 

   b. StationService - 역 관리 기능 수행 클래스 (역 등록/삭제)

   c. LineService - 노선 관리 기능 수행 클래스 (노선 등록/삭제)

   d. IntervalService - 구간 관리 기능 수행 클래스 (구간 등록/삭제)

4. **util 패키지** - 필요 유틸리티 

   a. InputValidator - 사용자 입력 검증을 담당하는 클래스

   <br>

## 🛠 상세 기능

1. 메인 화면 구현

   - (1) 역 관리, (2) 노선 관리, (3) 구간 관리, (4) 지하철 노선도 출력, (Q) 종료

   - **[사용자 입력]**: 사용자 입력으로 위 5개 보기 중 하나 선택

     - 발생 가능한 예외 상황:

       a. 숫자가 아닌 입력 (Q or q 제외)

       b.  보기 이외의 숫자 (1-4 이외의 숫자)

2. 첫번째 기능:  역 관리

   A. 역 등록

   - **[사용자 입력]:** 새로운 역 이름 등록
     - 발생 가능한 예외 상황:
       1. 2글자 이하
       2. 이름 중간에 공백 입력
       3. 중복된 역 이름
   - 입력된 역을 repository에 추가하고 [INFO]로 시작하는 안내 메세지 출력
   
B. 역 삭제
   
- **[사용자 입력]:** 삭제할 역 이름 입력:
     - 발생 가능한 예외 상황:
       1. 존재하지 않는 역 이름 입력
       2. 노선에 등록된 역 이름 입력
   - 입력된 역을 repository에서 삭제하고 [INFO]로 시작하는 안내 메세지 출력
   
C. 역 조회
   
- 역 repository에 있는 정보 출력
     - [INFO]로 시작하여 역 이름을 출력
   
3. 두번째 기능: 노선 관리

   A. 노선 등록

   - **[사용자 입력]**: 새로운 노선 이름 등록
     - 발생 가능한 예외 상황:
       1. 2글자 이하
       2. 노선 이름 중간에 공백 입력
       3. 중복된 노선 이름
   - **[사용자 입력]:** 노선 상행 종점역과 하행 종점역 입력
     - 노선에 상행 종점역과 하행 종점역을 추가할 때, 해당 역에 생성된 노선을 추가
       - 특정 역을 기준으로 해당 역이 속해있는 노선을 관리하기 위함
     - 발생 가능한 예외 상황:
       1. 존재하지 않는 역 이름 입력
   - 입력된 노선을 repository에 추가하고 [INFO]로 시작하는 안내 메세지 출력

   B. 노선 삭제

   - **[사용자 입력]** : 삭제할 노선 이름 입력
     - 발생 가능한 예외 상황
       1. 존재하지 않는 노선 이름 입력
     - 입력된 노선을 repository에서 삭제하고 [INFO]로 시작하는 안내 메세지 출력

   C. 노선 조회

   - 노선 repository에 있는 노선 정보 출력
     - [INFO]로 시작하여 노선 이름을 출력

4. 세번째 기능: 구간 관리

   A. 구간 등록

   - **[사용자 입력]**: 구간을 등록할 노선 이름을 입력

     - 발생 가능한 예외 상황:
       1. 존재하지 않는 노선 이름 입력

   - **[사용자 입력]**: 구간에 추가할 역 이름 입력

     - 발생 가능한 예외 상황:
       1. 존재하지 않는 역 이름 입력
       2. 이미 해당 노선에 입력되어 있는 역 이름 입력

   - **[사용자 입력]**: 구간 순서 입력 (순서는 1부터 시작)

     - 발생 가능한 예외 상황:

       1. 양의 정수 이외의 입력 (문자, 소수, 음의 정수 등등)
       2. `노선에 등록된 역의 개수 + 1` 보다 큰 범위의 순서 숫자가 입력되는 경우

       - 역과 역 사이에 구간을 추가할 수 있으므로 위 범위를 넘어가면 out of bound

   - 해당 구간에 역을 등록하고 [INFO]로 시작하는 안내 메세지 출력

   B. 구간 삭제

   - **[사용자 입력]:** 삭제할 구간의 노선 이름을 입력
     - 발생 가능한 예외 상황:
       1. 존재하지 않는 노선 이름 입력
   - **[사용자 입력]:** 구간에서 삭제할 역 이름 입력
     - 발생 가능한 예외 상황:
       1. 존재하지 않는 역 이름 입력
       2. 해당 구간에 포함된 역이 2개 이하인 경우 역 삭제 불가
   - 해당 구간에서 역을 삭제하고 [INFO]로 시작하는 안내 메세지 출력

5. 네번째 기능: 지하철 노선도 출력
   * 모든 출력은 [INFO]로 시작
     * 각 노선도 이름
     * 각 노선도에 포함된 역 이름을 상행 부터 하행 종점까지 차례로 출력

6. 다섯번째 기능: 프로그램 종료
   
* 메인화면에서 들어오는 사용자 입력에서 Q가 입력 되었을 때 종료 안내 문구 출력 후 프로그램 종료
  
7. 에러 메세지:

   * 위에서 언급된 예외 상황에 [ERROR]로 시작하는 에러 메세지 문구를 출력

   * 에러 메세지 문구 출력 후, 메뉴를 재선택 하도록 예외 복구를 실행

     <br>

## 🚀 기능 요구사항

### 초기 설정 
- 프로그램 시작 시 역, 노선 등 필요한 정보를 미리 셋팅할 수 있다.

> 아래의 사전 등록 정보로 반드시 초기 설정을 하기
>
```
 1. 지하철역으로 교대역, 강남역, 역삼역, 남부터미널역, 양재역, 양재시민의숲역, 매봉역이 등록되어 있다.
 2. 지하철 노선으로 2호선, 3호선, 신분당선이 등록되어 있다.
 3. 노선에 역이 아래와 같이 등록되어 있다.(왼쪽 끝이 상행 종점)
   - 2호선: 교대역 - 강남역 - 역삼역
   - 3호선: 교대역 - 남부터미널역 - 양재역 - 매봉역
   - 신분당선: 강남역 - 양재역 - 양재시민의숲역
```

### 지하철 역 관련 기능
- 지하철 역을 등록하고 삭제할 수 있다. (단, 노선에 등록된 역은 삭제할 수 없다)
- 중복된 지하철 역 이름이 등록될 수 없다.
- 역 이름은 2자 이상, 5자 이하만 가능하다.
- 지하철 역의 목록을 조회할 수 있다.

### 지하철 노선 관련 기능
- 지하철 노선을 등록하고 삭제할 수 있다.
- 중복된 지하철 노선 이름이 등록될 수 없다.
- 노선 이름은 2자 이상, 5자 이하만 가능하다.
- 노선 등록 시 상행 종점역과 하행 종점역을 입력받는다. 
- 지하철 노선의 목록을 조회할 수 있다.

### 지하철 구간 추가 기능
- 지하철 노선에 구간을 추가하는 기능은 노선에 역을 추가하는 기능이라고도 할 수 있다.
  - 역과 역사이를 구간이라 하고 이 구간들의 모음이 노선이다.  
- 하나의 역은 여러개의 노선에 추가될 수 있다.
- 역과 역 사이에 새로운 역이 추가 될 수 있다.
- 노선에서 갈래길은 생길 수 없다.

<img src="image/section1.png" width="500">

### 지하철 구간 삭제 기능
- 노선에 등록된 역을 제거할 수 있다.
- 종점을 제거할 경우 다음 역이 종점이 된다.
- 노선에 포함된 역이 두개 이하일 때는 역을 제거할 수 없다.

<img src="image/section2.png" width="500">

### 지하철 노선에 등록된 역 조회 기능
- 노선의 상행 종점부터 하행 종점까지 연결된 순서대로 역 목록을 조회할 수 있다.

- `프로그래밍 실행 결과 예시`를 참고하여 입출력을 구현한다.

- 에러 발생 시 `[ERROR]`를 붙여서 출력한다. (에러의 문구는 자유롭게 작성한다.)

## 🎱 프로그래밍 요구사항
- 자바 코드 컨벤션을 지키면서 프로그래밍한다.
  - 기본적으로 [Google Java Style Guide](https://google.github.io/styleguide/javaguide.html)을 원칙으로 한다.
  - 단, 들여쓰기는 '2 spaces'가 아닌 '4 spaces'로 한다.
- indent(인덴트, 들여쓰기) depth를 3이 넘지 않도록 구현한다. 2까지만 허용한다.
  - 예를 들어 while문 안에 if문이 있으면 들여쓰기는 2이다.
  - 힌트: indent(인덴트, 들여쓰기) depth를 줄이는 좋은 방법은 함수(또는 메소드)를 분리하면 된다.
- 3항 연산자를 쓰지 않는다.
- 함수(또는 메소드)의 길이가 15라인을 넘어가지 않도록 구현한다.
  - 함수(또는 메소드)가 한 가지 일만 잘 하도록 구현한다.
- else 예약어를 쓰지 않는다.
  - 힌트: if 조건절에서 값을 return하는 방식으로 구현하면 else를 사용하지 않아도 된다.
  - else를 쓰지 말라고 하니 switch/case로 구현하는 경우가 있는데 switch/case도 허용하지 않는다.
- 프로그래밍 요구사항에서 별도로 변경 불가 안내가 없는 경우 파일 수정과 패키지 이동을 자유롭게 할 수 있다.
- 예외 상황 시 에러 문구를 출력해야 한다. 단, 에러 문구는 `[ERROR]` 로 시작해야 한다.

### 프로그래밍 요구사항 - Application
- Application 클래스를 활용해 구현해야 한다.
- Application의 패키지 구조는 변경하지 않는다.
- 주석을 참고하여 구현할 수 있으며 주석대로 구현하지 않아도 되고 삭제해도 무관하다. 
```java
public class Application {
    public static void main(String[] args) {
        ...
    }
}
```

### 프로그래밍 요구사항 - Station, Line
- Station, Line 클래스를 활용하여 지하철역과 노선을 구현해야 한다.
- 제공하는 각 클래스의 기본 생성자를 추가할 수 없다.
- 필드(인스턴스 변수)인 name의 접근 제어자 private을 변경할 수 없다.
- 가능하면 setter 메소드(ex. setXXX)를 추가하지 않고 구현한다.

```java
public class Station {
    private String name;

    public Station(String name) {
        this.name = name;
    }

    public String getName() {
        return name;
    }

    // 추가 기능 구현
}

```

### 프로그래밍 요구사항 - StationRepository, LineRepository
- Station과 Line의 상태를 저장할 수 있는 StationRepository, LineRepository를 제공한다.
- 필요 시 StationRepository, LineRepository 이 외 추가로 Repository를 만들 수 있다.
- 추가로 생성되는 객체에 대해서 XXXRepository 네이밍으로 저장 클래스를 추가할 수 있다.
- 객체들의 상태를 관리하기 위해서 XXXRepository 클래스를 활용해 저장 로직을 구현해야 한다.
- 필요에 따라 자유롭게 수정이 가능하다.

```java
public class StationRepository {
    private static final List<Station> stations = new ArrayList<>();

    public static List<Station> stations() {
        return Collections.unmodifiableList(stations);
    }

    public static void addStation(Station station) {
        stations.add(station);
    }

    public static boolean deleteStation(String name) {
        return stations.removeIf(station -> Objects.equals(station.getName(), name));
    }
}
```
